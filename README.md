# Comparación entre Programación Tradicional y POO en Python

## Introducción
En este trabajo desarrollé un programa en Python para calcular el promedio semanal del clima. La actividad se realizó utilizando dos enfoques diferentes: programación tradicional y programación orientada a objetos (POO). El objetivo principal es comprender las diferencias entre ambos paradigmas y analizar cómo se puede resolver un mismo problema aplicando distintas formas de programación.

---

## Programación Tradicional
Para la programación tradicional implementé una solución basada en funciones. Primero creé una función para ingresar las temperaturas diarias y luego otra función encargada de calcular el promedio semanal.

Este enfoque es sencillo y fácil de entender, especialmente cuando se trabaja con programas pequeños. Sin embargo, a medida que el programa crece, puede resultar más difícil de mantener, ya que toda la lógica depende del uso correcto de las funciones.

El archivo que contiene esta solución es:
- `tradicional.py`

---

## Programación Orientada a Objetos (POO)
En la solución orientada a objetos utilicé una clase para representar la información del clima semanal. Dentro de la clase definí métodos para ingresar las temperaturas y calcular el promedio.

En este código apliqué conceptos importantes de la programación orientada a objetos como:
- Encapsulamiento, para proteger los datos.
- Herencia, mediante el uso de una clase base.
- Polimorfismo, al redefinir el método que calcula el promedio.

Este enfoque permite que el código esté mejor organizado y sea más fácil de mantener o ampliar en el futuro.

El archivo que contiene esta solución es:
- `poo.py`

---

## Comparación
Ambos enfoques permiten obtener el promedio semanal del clima, pero presentan diferencias claras. La programación tradicional es más directa y fácil de implementar, mientras que la programación orientada a objetos ofrece una mejor estructura y organización del código, especialmente para proyectos más grandes.

---

## Conclusión
Con la realización de esta práctica pude comprender mejor las diferencias entre la programación tradicional y la programación orientada a objetos. Aunque ambos métodos son válidos, considero que la POO resulta más adecuada cuando se busca un código más ordenado, reutilizable y escalable.
